# 午餐随机抽取系统 - 完整开发方案
## 1. 项目概述
### 1.1 项目背景
解决团队日常午餐选择困难的问题，通过网页形式提供菜品添加、权重设置、随机抽取、动态展示及海报生成功能，提升选择过程的趣味性和互动性。

### 1.2 核心目标
- 支持多人协作添加菜品，并为菜品设置随机权重（权重越高，被抽中概率越大）
- 实现视觉化、趣味性的随机抽取动画效果
- 抽取结果生成个性化、唯一的分享海报
- 网页可部署至服务器，支持团队成员随时访问使用

## 2. 功能需求
### 2.1 菜品管理模块
| 功能点 | 详细描述 |
|--------|----------|
| 菜品添加 | 支持输入菜品名称、分类（如快餐/火锅/面食等）、权重值（1-10分，默认5分）、备注（可选，如推荐店铺） |
| 菜品列表 | 展示所有已添加菜品，包含名称、分类、权重、添加人、添加时间 |
| 菜品编辑/删除 | 支持修改菜品信息或删除无效菜品（可增加简单权限控制，如仅添加人可编辑） |
| 权重可视化 | 菜品列表中通过进度条/数字标识权重大小，直观展示抽中概率 |

### 2.2 随机抽取模块
| 功能点 | 详细描述 |
|--------|----------|
| 抽取触发 | 点击「开始随机」按钮启动抽取流程 |
| 动态展示效果 | 抽取过程中实现「滚动闪烁」式动画：快速切换菜品名称，速度由快到慢最终停在结果上，伴随轻微的视觉特效（如缩放、变色） |
| 概率逻辑 | 严格按照菜品权重计算抽中概率（如权重总和为S，菜品A权重为W，则概率=W/S） |
| 抽取结果展示 | 放大展示最终抽中的菜品，包含名称、分类、备注等信息，搭配庆祝类动效（如礼花、弹窗） |

### 2.3 海报生成模块
| 功能点 | 详细描述 |
|--------|----------|
| 自动生成海报 | 抽取完成后，一键生成包含以下信息的海报：<br>- 菜品名称（醒目展示）<br>- 抽取时间<br>- 随机幸运文案（如「今日干饭首选！」「天选打工人的午餐」）<br>- 团队专属logo/背景（可配置）<br>- 唯一标识（如随机编号/抽取次数） |
| 海报样式 | 支持1-2种固定模板，配色鲜明，适配移动端分享 |
| 海报操作 | 支持海报下载（PNG格式）、一键分享（复制图片/链接） |

### 2.4 基础功能
| 功能点 | 详细描述 |
|--------|----------|
| 数据存储 | 菜品信息持久化存储（推荐使用轻量数据库如SQLite，或JSON文件） |
| 跨端访问 | 网页适配PC端和移动端，支持主流浏览器（Chrome/Firefox/Edge） |
| 部署要求 | 代码轻量化，可直接部署至Linux/Windows服务器，依赖少易维护 |
